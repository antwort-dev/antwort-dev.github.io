OpenResponses API

The only open-source server-side implementation of the OpenResponses standard. Any OpenAI SDK (Python, Node, Go, Rust) works without modification. No vendor lock-in. No proprietary API to learn.

Secure by Default

Every code execution runs in a gVisor-isolated Kubernetes Pod. Network deny-all. mTLS with SPIFFE/SPIRE. Audit logging for every tool call. Security is the architecture, not an option.

Kubernetes Native

Deploy with kubectl apply. Scale with HPA. Monitor with Prometheus. Multi-tenant from day one. Designed exclusively for Kubernetes, with no standalone mode.

Features

Agentic Loop

Agents that act, not just answer

Multi-turn reasoning with concurrent tool execution. Your agent calls tools, gets results, reasons, and calls more tools until the job is done.

Multi-Provider

Bring your own model

vLLM, LiteLLM, Ollama, or any OpenAI-compatible endpoint. Switch providers without changing your client code. Zero vendor lock-in.

MCP Tools

Connect to any tool server

Model Context Protocol integration with OAuth, tool discovery, and automatic routing. Your agent can use any MCP-compatible tool server.

Multi-Tenant Auth

Enterprise-ready from day one

JWT/OIDC and API key authentication with per-tenant isolation. Rate limiting by service tier. Every response is scoped to the authenticated user.

Conversation Memory

Agents that remember

PostgreSQL-backed conversation persistence with response chaining. Full history reconstruction across multi-turn interactions.

SSE Streaming

Real-time token streaming

Server-Sent Events with complete lifecycle events. Watch your agent think, call tools, and respond in real time.

Web Search

Current information retrieval

Built-in web search via SearXNG integration. Your agent can find and reference current information without external tool setup.

Observability

Know what your agents are doing

Prometheus metrics for request latency, token usage, tool execution, and streaming connections. OpenTelemetry GenAI semantic conventions.

Production Deployment

Ship to any Kubernetes cluster

Kustomize overlays for dev, production, and OpenShift. Health probes, HPA, Pod disruption budgets included.

Sandbox Execution Coming Soon

Code runs in Kubernetes, not on your laptop

Every code_interpreter call executes in a gVisor-isolated Pod with network deny-all, resource limits, and audit logging. Powered by agent-sandbox CRDs.

Agent Profiles Coming Soon

Server-side SOUL.md

Reusable agent configurations with instructions, tools, model preferences, and sandbox policies. Manage agents centrally, reference by ID.

RAG & Knowledge Coming Soon

Give your agent a knowledge base

Vector stores with pgvector, document ingestion, and file_search as a built-in tool. Your agent retrieves relevant context automatically.

Proactive Scheduling Coming Soon

Agents that work while you sleep

Cron triggers, webhook triggers, and completion hooks. Your agent scans alerts at 3am and files tickets before you wake up.

Delivery Channels Coming Soon

Results where you need them

Push agent output to Slack, Teams, email, or webhooks. Combined with scheduling, agents become autonomous workers.

Tool Registry Coming Soon

Curated, security-audited tools

Managed tool catalog with per-tenant permissions. MCP servers registered with security review. Every invocation logged.
